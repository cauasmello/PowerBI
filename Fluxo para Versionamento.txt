Fluxo correto para o dia a dia:
- Salvar as alterações;
- "git status"; 
- "git add .";
- "git commit -m "Descrição das alterações" "

-------------------------------------------------------------------------------------
Ver o Histórico do Projeto:
- "git log --oneline --decorate --graph --all"
-------------------------------------------------------------------------------------
Commitar:
- "git push -u origin master"
-------------------------------------------------------------------------------------
Branch:
- Muito importante para conseguir ter acesso a versões antigas.
------------------------------------------------------------------------------------
Criar uma branch a partir de alguma versão "antiga:"
-"git checkout -b hotfix/voltar-versao-antiga <hash do commit>"
------------------------------------------------------------------------------------
Restaurar um arquivo específico para uma versão antiga:
-"git checkout -b hotfix/voltar-versao-antiga <hash do commit>"
------------------------------------------------------------------------------------
Desfazer um Commit:
-"git revert <hash do commit>"
------------------------------------------------------------------------------------
Mudar de Branch:
- "git switch nome-da-branch"

Criar uma nova Branch e já ir pra ela:
- "git switch -c minha-nova-branch"

OBS: O GitHub pode ser usado facilmente junto com o PowerBI para controle de versões.
